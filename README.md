# Hamster-BOCRA-website

Website for BOCRA hackathon, includes frontend and backend.

## Setup: Environment Variables

This project uses **Supabase** for authentication, database, and storage. You need to configure environment variables for both the frontend and backend.

### Frontend Setup

Create `Frontend/.env` (copy from `Frontend/.env.example`):

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=your-publishable-key
```

### Backend Setup

Create `backend/.env` (copy from `backend/.env.example`):

```
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SECRET_KEY=your-secret-key
```

**Where to get these values:**
1. Go to your [Supabase Dashboard](https://app.supabase.com)
2. Select your project → Settings → API
3. Copy the values into your `.env` files

**Key Security Breakdown:**
- **VITE_SUPABASE_PUBLISHABLE_KEY**: Safe to expose in frontend code. Used for public operations like authentication.
- **SUPABASE_SECRET_KEY**: Backend-only. Gives full administrative access. Never expose to the frontend.

**⚠️ Security:**
- `.env` files are **never** committed to git (see `.gitignore`)
- The secret key must never reach the frontend or client code
- Keep your `.env` files private and secure

See [backend/README.md](backend/README.md) for detailed backend setup.

## Run the Frontend

1. Open a terminal in the project root.
2. Go to the frontend directory:

```bash
cd Frontend
```

3. Install dependencies:

```bash
npm install
```

4. Start the development server:

```bash
npm run dev
```

5. Open your browser at:

```text
http://localhost:8080
```

## Useful Frontend Commands

- Build for production:

```bash
npm run build
```

- Preview production build locally:

```bash
npm run preview
```

- Run lint checks:

```bash
npm run lint
```

## Tech Stack

### Frontend
- **Framework**: React 18 with TypeScript
- **Build Tool**: Vite
- **Styling**: TailwindCSS
- **Routing**: React Router v6
- **Backend Communication**: Axios
- **Testing**: Vitest + React Testing Library
- **Package Manager**: npm/Bun

### Backend
- **Framework**: FastAPI (Python 3.12)
- **ORM & Database**: Supabase PostgreSQL
- **Authentication**: Supabase Auth
- **Validation**: Pydantic v2
- **Testing**: pytest
- **API Documentation**: OpenAPI (Swagger) - auto-generated by FastAPI

### Database & Storage
- **Database**: Supabase PostgreSQL
- **Authentication**: Supabase Auth (JWT-based)
- **File Storage**: Supabase Storage (for profile photos, documents, etc.)
- **Real-time**: Supabase Realtime for live updates

### DevOps & Infrastructure
- **Backend Hosting**: Railway
- **Frontend Hosting**: Vercel
- **Version Control**: Git/GitHub

## Deployment

### Backend Deployment (Railway)

The backend is deployed on [Railway.app](https://railway.app). Configuration is defined in `backend/railway.toml`.

**Deployment Process:**
1. Push changes to the main branch
2. Railway automatically builds and deploys the FastAPI application
3. Environment variables (SUPABASE_URL, SUPABASE_SECRET_KEY) are configured in Railway dashboard

**Key Configuration:**
- Production database: Connected to Supabase PostgreSQL
- Auto-scaling: Railway handles load balancing
- Logs: Available in Railway dashboard for debugging

**Running Tests Before Deployment:**
```bash
cd backend
python -m pytest tests/ -q
```

### Frontend Deployment (Vercel)

The frontend is deployed on [Vercel](https://vercel.com). Configuration is defined in `vercel.json`.

**Deployment Process:**
1. Push changes to the main branch
2. Vercel automatically detects the Vite React project
3. Builds the project using `npm run build`
4. Deploys to Vercel's CDN with automatic SSL certificates
5. Environment variables (VITE_SUPABASE_URL, VITE_SUPABASE_PUBLISHABLE_KEY) are configured in Vercel project settings

**Key Configuration:**
- Build Command: `npm run build`
- Output Directory: `dist`
- Node Version: 18.x or later
- Preview deployments for all pull requests

**Running Tests Before Deployment:**
```bash
cd Frontend
npm run test
```

### Environment Variables in Production

Both backends and frontend read environment variables from their respective hosting platforms:

**Vercel (Frontend):**
- `VITE_SUPABASE_URL`: Supabase project URL
- `VITE_SUPABASE_PUBLISHABLE_KEY`: Supabase publishable API key (safe for frontend)

**Railway (Backend):**
- `SUPABASE_URL`: Supabase project URL
- `SUPABASE_SECRET_KEY`: Supabase secret API key (backend-only)

Never commit `.env` files or secret keys to version control.
