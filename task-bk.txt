C) Storage Buckets
------------------
Create these Supabase storage buckets:

- application-documents   (private, 10MB limit, PDF/JPG/PNG only)
- profile-photos          (private, 5MB limit, JPG/PNG only)
- evidence-uploads        (private, 10MB limit, PDF/JPG/PNG/MP4)
- public-documents        (public, 20MB limit, PDF only — regulations, forms, reports)

Set RLS policies:
- Users can only read/write their own files in private buckets
- Admins can read all files
- Public bucket is readable by everyone


D) Row Level Security (RLS) Policies
-------------------------------------
Enable RLS on ALL tables. Key policies:

profiles:
  - Users can read and update their own profile
  - Admins can read all profiles

applications:
  - Users can CRUD their own applications
  - Admins can read all, update status/notes/decision fields

documents:
  - Users can upload and view their own documents
  - Admins can view all documents

complaints:
  - Users can create and view their own complaints
  - Admins can read all, update status and response

payments:
  - Users can view their own payments
  - Admins can view all payments

news, telecom_stats, regulatory_decisions:
  - Anyone can read published/public records
  - Only admins can create/update/delete

audit_log:
  - Only admins can read
  - System inserts only (no user writes)

STEP 4: SECURITY (Compliance C, D, E)
---------------------------------------
These are NON-NEGOTIABLE. Implement from day one.

Authentication & Access:
  - Supabase Auth handles JWT tokens — validate on every backend request
  - Role-based middleware: check profile.role before admin endpoints
  - Rate limiting on auth endpoints (5 attempts per minute)
  - CORS locked to Vercel domain + localhost

Data Protection:
  - Supabase encrypts data at rest (Postgres) and in transit (TLS)
  - Never return sensitive fields (id_number, password hash) in list endpoints
  - Input validation on ALL endpoints using Pydantic models
  - Sanitize file uploads: validate MIME type, scan file extension
  - Data minimisation: forms only collect what's necessary (compliance C)

Audit Trail:
  - Log EVERY state change to audit_log table
  - Include: user_id, action, resource, IP address, timestamp
  - Audit log is append-only — no update or delete allowed

Admin Security:
  - Tiered admin permissions if time allows (compliance E)
  - Suspicious login detection: flag if same account logs in from new IP
  - Consider 2FA for admin accounts (Supabase supports TOTP) — stretch goal

Backups:
  - Supabase handles daily automatic backups on paid plans
  - Document the backup strategy in the README
